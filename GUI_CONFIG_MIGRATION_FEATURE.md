# 智能体配置迁移功能 - GUI界面说明

## 功能位置

在Neo Agent GUI应用的右侧控制面板中，"系统设置"部分新增了两个按钮：

### 按钮布局

```
┌─────────────────────────────────┐
│      系统设置                    │
├─────────────────────────────────┤
│                                 │
│  📤 导出智能体配置               │
│                                 │
│  📥 导入智能体配置               │
│                                 │
│  ♻️ 重新加载代理                 │
│                                 │
│  ℹ️ 关于                        │
│                                 │
└─────────────────────────────────┘
```

## 功能说明

### 📤 导出智能体配置
点击此按钮将打开文件保存对话框，允许用户选择保存位置并导出当前智能体的完整配置到JSON文件。

**导出内容包括：**
- 环境变量（.env文件中的所有配置）
- 环境描述（所有环境及其物体、连接关系）
- 基础知识库（所有基础事实）
- 智能体表达风格（个性化表达方式）

**默认文件名：** `agent_config_YYYYMMDD_HHMMSS.json`

### 📥 导入智能体配置
点击此按钮将打开文件选择对话框，允许用户选择配置文件并导入到当前智能体。

**导入模式：**
- **覆盖模式**：替换已存在的配置项
- **追加模式**：只添加新配置，保留现有配置

**导入后建议**：点击"♻️ 重新加载代理"以应用新配置

## 使用流程

### 导出配置
1. 点击 "📤 导出智能体配置"
2. 在文件保存对话框中选择保存位置
3. 输入文件名（或使用默认名称）
4. 点击"保存"
5. 看到成功提示消息

### 导入配置
1. 点击 "📥 导入智能体配置"
2. 在文件选择对话框中选择要导入的JSON文件
3. 在弹出的对话框中选择导入模式（覆盖/追加）
4. 点击"打开"开始导入
5. 看到成功提示消息
6. 点击 "♻️ 重新加载代理" 应用新配置

## 配置文件示例

导出的JSON文件结构清晰，易于阅读和编辑：

```json
{
  "version": "1.0",
  "export_time": "2026-01-09T18:00:00.000000",
  "env_config": {
    "CHARACTER_NAME": "小可",
    "CHARACTER_AGE": "18",
    "MODEL_NAME": "deepseek-ai/DeepSeek-V3"
  },
  "environments": [...],
  "base_knowledge": [...],
  "agent_expressions": [...]
}
```

## 安全提示

⚠️ **重要提醒：**
- 导出的配置文件可能包含敏感信息（API密钥等）
- 请妥善保管配置文件
- 分享前务必删除敏感信息
- 建议定期备份配置文件
